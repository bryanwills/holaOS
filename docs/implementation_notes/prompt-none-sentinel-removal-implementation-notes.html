Prompt None Sentinel Removal Implementation Notes

Design decisions

  • Removed synthetic Attachments: none. and Image inputs: none. prompt lines entirely instead of replacing them with shorter placeholders.
  • Kept all positive attachment and image sections unchanged so prompts still surface real attached content and referenced image URLs.

Deviations

  • No spec deviation. The change follows the request to omit no-op attachment and image-input prompt text.

Tradeoffs

  • Removing the sentinel lines makes prompts smaller and avoids priming the model with attachment and image vocabulary when no such inputs exist.
  • The prompt no longer explicitly states absence, so any downstream consumer that relied on that exact text must use attachment arrays rather than prompt string matching.

Open questions

  • None at implementation time.
