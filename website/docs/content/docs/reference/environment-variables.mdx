holaOS uses environment variables at several layers:

- desktop and standalone runtime startup
- workspace app startup inside workspaces
- bridge and integration helpers inside workspace apps

The tables below focus on the names that are actively exercised in the current repo and runtime contracts. Release-only signing and CI publishing variables are intentionally omitted.

## Desktop app and runtime staging

| Variable | Purpose |
| --- | --- |
| `HOLABOSS_AUTH_BASE_URL` | Better Auth / Hono base URL consumed by the desktop and injected into the embedded runtime for authenticated runtime-side calls. |
| `HOLABOSS_AUTH_SIGN_IN_URL` | Hosted sign-in page URL opened by the desktop auth flow. |
| `HOLABOSS_BACKEND_BASE_URL` | Desktop/backend base URL used by local dev validation and remote bridge fallback. |
| `HOLABOSS_DESKTOP_CONTROL_PLANE_BASE_URL` | Optional override for the desktop control-plane gateway base URL. |
| `HOLABOSS_INTERNAL_DEV` | Enables local or non-production backend overrides in desktop development paths. |
| `HOLABOSS_DESKTOP_USER_DATA_PATH` | Explicit Electron `userData` directory path. Also changes the derived embedded runtime port when `HOLABOSS_RUNTIME_API_PORT` is unset. |
| `HOLABOSS_DESKTOP_USER_DATA_DIR` | Directory name used under Electron app-data when `HOLABOSS_DESKTOP_USER_DATA_PATH` is not set. |
| `HOLABOSS_RUNTIME_API_PORT` | Explicit embedded desktop runtime port override. If unset, the desktop derives the port from the resolved `userData` path and rejects `5060`. |
| `HOLABOSS_RUNTIME_ROOT` | Override for the packaged runtime bundle root used by the Electron main process. |
| `HOLABOSS_RUNTIME_PLATFORM` | Override for the target platform used by runtime staging and packaging scripts. |
| `HOLABOSS_OSS_ROOT` / `HOLABOSS_RUNTIME_REPO_ROOT` | Optional repo-root override used by `prepare-runtime:local` when the script runs from another checkout. |
| `HOLABOSS_RUNTIME_DIR` | Explicit unpacked runtime-bundle source used by staging scripts. |
| `HOLABOSS_RUNTIME_TARBALL` | Explicit runtime-bundle tarball source used by staging scripts. |
| `HOLABOSS_RUNTIME_BUNDLE_URL` | Explicit runtime-bundle download URL used by staging scripts. |

## Runtime bootstrap and model routing

| Variable | Purpose |
| --- | --- |
| `HB_SANDBOX_ROOT` | Root directory for runtime workspace and state data. |
| `SANDBOX_AGENT_BIND_HOST` | Host interface the runtime binds to. |
| `SANDBOX_AGENT_BIND_PORT` | Runtime API port. |
| `SANDBOX_RUNTIME_API_PORT` | Preferred runtime HTTP port env for standalone runtime startup and runtime-side URL derivation. |
| `SANDBOX_AGENT_HARNESS` | Harness selector, typically `pi`. |
| `HOLABOSS_RUNTIME_WORKFLOW_BACKEND` | Workflow backend selector used by the runtime. |
| `HOLABOSS_HOST_STATE_DB_PATH` | Preferred override for the sandbox-global host-state SQLite path. |
| `HOLABOSS_CONTROL_PLANE_DB_PATH` | Optional override for the control-plane SQLite path. |
| `HOLABOSS_RUNTIME_DB_PATH` | Legacy alias for `HOLABOSS_HOST_STATE_DB_PATH`. |
| `HOLABOSS_RUNTIME_CONFIG_PATH` | Optional override for `runtime-config.json`. |
| `HOLABOSS_DEFAULT_MODEL` | Default model id used when one is not selected explicitly. |
| `SANDBOX_AGENT_DEFAULT_MODEL` | Fallback default model when `HOLABOSS_DEFAULT_MODEL` is not set. |
| `HOLABOSS_MODEL_PROXY_BASE_URL` | Proxy base URL for model routing. |
| `HOLABOSS_SANDBOX_AUTH_TOKEN` | Auth token used when talking to the proxy path. |
| `HOLABOSS_AUTH_COOKIE` | Desktop-injected auth cookie used by runtime-side Hono and Composio routes. |
| `HOLABOSS_AUTH_BEARER_TOKEN` | Desktop-injected bearer token used by runtime-side Composio internal calls. |
| `PROACTIVE_ENABLE_REMOTE_BRIDGE` | Enables the remote bridge path when set to `1`. |
| `PROACTIVE_BRIDGE_BASE_URL` | Base URL for the proactive bridge. |

## Module app bridge and workspace integration

| Variable | Purpose |
| --- | --- |
| `HOLABOSS_APP_GRANT` | Grant token used by the bridge SDK to proxy provider requests. |
| `HOLABOSS_WORKSPACE_ID` | Current workspace id for output writes and workspace-scoped actions. |
| `HOLABOSS_INTEGRATION_BROKER_URL` | Explicit broker URL override for integration proxy calls. |
| `WORKSPACE_API_URL` | Base URL for publishing workspace outputs. |
| `WORKSPACE_<PROVIDER>_INTEGRATION_ID` | Runtime-injected connection id for a resolved integration binding, for example `WORKSPACE_GMAIL_INTEGRATION_ID`. |
| `SANDBOX_RUNTIME_API_PORT` | Runtime API port used by bridge helpers to resolve local integration URLs. |
| `SANDBOX_AGENT_BIND_PORT` | Fallback port used when the runtime API port is not available. |
| `HOLABOSS_USER_ID` | User id passed into workspace apps that need a holaOS identity. |

## Legacy compatibility

| Variable | Purpose |
| --- | --- |
| `PLATFORM_INTEGRATION_TOKEN` | Legacy compatibility variable. It is not part of the current runtime injection path for workspace apps. |

## How the bridge resolves URLs

The bridge helper used by workspace apps follows a simple resolution order:

1. use `HOLABOSS_INTEGRATION_BROKER_URL` if it is set
2. otherwise derive the integration broker from the runtime port
3. derive the workspace API URL from the broker URL when needed

If those variables are missing, the bridge helpers fall back to a no-op path rather than crashing the app.

## Where to set them

- desktop values usually live in the desktop env file or the runtime config file
- runtime startup values are often injected by the launcher or packaged bundle
- workspace app values are injected by the runtime when the app starts, or declared as expected inputs in `app.runtime.yaml`

<Callout type="warn">
Not every environment variable is required in every mode. Only set the ones that match the execution target you are actually running.
</Callout>
